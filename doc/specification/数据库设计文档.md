## 一、数据库设计原则

### 1.1 设计规范

1. **命名规范**
   - 数据库名：小写下划线，如 `rental_system`
   - 表名：小写下划线_模块，如 `sys_user`、`biz_house`
   - 字段名：小写下划线，如 `user_name`
   - 索引名：`idx_表名_字段名`、`uk_表名_字段名`
   - 外键名：`fk_表名_关联表名`

2. **字段规范**
   - 所有表必须有 `id`、`create_time`、`update_time`、`create_by`、`update_by`、`deleted` 字段
   - 字符串使用 `varchar`，根据实际需求设置长度
   - 金额使用 `decimal(10,2)`
   - 时间使用 `datetime`
   - 状态字段使用 `tinyint`

3. **索引规范**
   - 主键自动创建索引
   - 根据查询需求创建普通索引和唯一索引
   - 复合索引注意最左前缀原则
   - 单表索引数量不超过5个

4. **第三范式(3NF)**
   - 每个字段只存储单一数据
   - 非主键字段完全依赖于主键
   - 消除传递依赖

### 3.1 用户表 (sys_user)

| 字段名 | 类型 | 允许空 | 默认值 | 说明 |
|--------|------|--------|--------|------|
| id | bigint | NO | AUTO_INCREMENT | 主键 |
| phone | varchar(11) | NO | - | 手机号(唯一) |
| username | varchar(50) | YES | - | 用户名 |
| avatar | varchar(255) | YES | - | 头像URL |
| gender | tinyint | YES | 0 | 性别：0-未知，1-男，2-女 |
| email | varchar(100) | YES | - | 邮箱 |
| password | varchar(255) | YES | - | 密码(加密存储) |
| status | tinyint | NO | 1 | 状态：0-禁用，1-正常 |
| create_time | datetime | NO | CURRENT_TIMESTAMP | 创建时间 |
| update_time | datetime | NO | CURRENT_TIMESTAMP | 更新时间 |
| create_by | bigint | YES | - | 创建人 |
| update_by | bigint | YES | - | 更新人 |
| deleted | tinyint | NO | 0 | 删除标记：0-未删除，1-已删除 |

```sql
CREATE TABLE sys_user (
    id BIGINT NOT NULL AUTO_INCREMENT COMMENT '用户ID',
    phone VARCHAR(11) NOT NULL COMMENT '手机号',
    username VARCHAR(50) DEFAULT NULL COMMENT '用户名',
    avatar VARCHAR(255) DEFAULT NULL COMMENT '头像URL',
    gender TINYINT DEFAULT 0 COMMENT '性别：0-未知，1-男，2-女',
    email VARCHAR(100) DEFAULT NULL COMMENT '邮箱',
    password VARCHAR(255) DEFAULT NULL COMMENT '密码',
    status TINYINT NOT NULL DEFAULT 1 COMMENT '状态：0-禁用，1-正常',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    create_by BIGINT DEFAULT NULL COMMENT '创建人',
    update_by BIGINT DEFAULT NULL COMMENT '更新人',
    deleted TINYINT NOT NULL DEFAULT 0 COMMENT '删除标记：0-未删除，1-已删除',
    PRIMARY KEY (id),
    UNIQUE KEY uk_sys_user_phone (phone),
    KEY idx_sys_user_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

## 四、索引策略

### 4.1 索引设计原则

1. **主键索引**：自动在主键上创建唯一索引
2. **高频查询字段**：为 `WHERE`、`JOIN`、`ORDER BY` 频繁使用的字段创建索引
3. **复合索引**：将选择性高的字段放在前面
4. **避免过多索引**：每个表索引不超过5个

### 4.2 核心索引

| 表名 | 索引名称 | 索引字段 | 类型 | 说明 |
|------|----------|----------|------|------|
| sys_user | uk_sys_user_phone | phone | 唯一 | 手机号唯一约束 |
| biz_house | idx_biz_house_region | region_id | 普通 | 按区域查询 |
| biz_house | idx_biz_house_rent | rent | 普通 | 按价格排序 |

## 五、事务管理

### 5.1 事务传播行为

| 传播行为 | 说明 |
|----------|------|
| REQUIRED | 默认，当前有事务则加入，没有则创建新事务 |
| REQUIRES_NEW | 总是创建新事务，挂起当前事务 |
| NESTED | 如果当前有事务，则在嵌套事务中执行 |

### 5.2 事务使用规范

```java
@Service
public class OrderService {

    @Transactional(rollbackFor = Exception.class)
    public void createOrder(OrderDTO dto) {
        // 1. 校验房源状态
        HouseEntity house = houseMapper.selectById(dto.getHouseId());
        if (house.getStatus() != 0) {
            throw new BusinessException("房源不可租");
        }

        // 2. 创建订单
        OrderEntity order = new OrderEntity();
        order.setOrderNo(generateOrderNo());
        orderMapper.insert(order);

        // 3. 更新房源状态
        house.setStatus(1);
        houseMapper.updateById(house);
    }
}
```

## 六、数据备份

### 6.1 备份策略

| 备份类型 | 执行频率 | 保留时间 |
|----------|----------|----------|
| 全量备份 | 每周日凌晨2点 | 30天 |
| 增量备份 | 每天凌晨2点 | 7天 |
| 实时备份 | 开启binlog | - |

### 6.2 备份脚本

```bash
#!/bin/bash
# 全量备份脚本
BACKUP_DIR=/data/backup/mysql
DATE=$(date +%Y%m%d)

# 创建备份目录
mkdir -p ${BACKUP_DIR}/${DATE}

# 执行备份
mysqldump -hlocalhost -uroot -ppassword --single-transaction --routines --triggers rental_system > ${BACKUP_DIR}/${DATE}/full_backup.sql

# 压缩
gzip ${BACKUP_DIR}/${DATE}/full_backup.sql

# 删除30天前的备份
find ${BACKUP_DIR} -type d -mtime +30 -exec rm -rf {} \;
```

### 6.3 恢复脚本

```bash
#!/bin/bash
# 恢复脚本
BACKUP_FILE=$1
mysql -hlocalhost -uroot -ppassword rental_system < ${BACKUP_FILE}
```
