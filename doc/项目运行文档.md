## 1. 项目概述

本项目是一个租房系统，提供房源浏览、搜索、筛选、详情查看等功能，同时支持用户注册登录、个人中心管理等功能。

**项目结构**：
```
GreatCreate/
├── backend/          # 后端服务（Spring Boot）
├── frontend/         # 前端项目（Vue 3）
├── doc/              # 技术文档
└── sql/              # 数据库脚本
```

## 4. 后端服务部署

### 4.1 数据库准备

**创建数据库**：
```sql
CREATE DATABASE IF NOT EXISTS rental_db 
  DEFAULT CHARACTER SET utf8mb4 
  DEFAULT COLLATE utf8mb4_unicode_ci;

USE rental_db;
```

**导入初始化脚本**：
```bash
mysql -u root -p rental_db < sql/init.sql
```

### 4.2 配置文件修改

编辑 `backend/src/main/resources/application.yml`：

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/rental_db?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8
    username: your_username    # 修改为你的数据库用户名
    password: your_password    # 修改为你的数据库密码
    driver-class-name: com.mysql.cj.jdbc.Driver

  data:
    redis:
      host: localhost
      port: 6379
      password:                # Redis密码（如有）

jwt:
  secret: your-256-bit-secret-key-here    # 修改为安全的密钥
  access-token-expiration: 7200000         # 2小时
  refresh-token-expiration: 604800000      # 7天
```

### 4.3 启动后端服务

**开发模式**：
```bash
cd backend
mvn spring-boot:run
```

**生产打包**：
```bash
cd backend
mvn clean package
java -jar target/rental-backend-1.0.0.jar
```

### 4.4 验证服务

访问健康检查接口：
```
http://localhost:8080/api/health
```

预期响应：
```json
{
  "code": 200,
  "message": "OK",
  "data": null
}
```

## 6. Docker部署

### 6.1 Docker Compose（推荐）

创建 `docker-compose.yml`：
```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: rental-mysql
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: rental_db
    ports:
      - "3306:3306"
    volumes:
      - ./sql/init.sql:/docker-entrypoint-initdb.d/init.sql
      - mysql_data:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      timeout: 20s
      retries: 10

  redis:
    image: redis:7.0
    container_name: rental-redis
    ports:
      - "6379:6379"

  backend:
    build: ./backend
    container_name: rental-backend
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_started
    environment:
      - SPRING_DATASOURCE_URL=jdbc:mysql://mysql:3306/rental_db
      - SPRING_DATASOURCE_USERNAME=root
      - SPRING_DATASOURCE_PASSWORD=root
      - SPRING_DATA_REDIS_HOST=redis
    ports:
      - "8080:8080"

  frontend:
    build: ./frontend
    container_name: rental-frontend
    depends_on:
      - backend
    ports:
      - "3000:80"

volumes:
  mysql_data:
```

**启动服务**：
```bash
docker-compose up -d
```

### 6.2 单独构建

**后端镜像**：
```bash
cd backend
docker build -t rental-backend .
docker run -p 8080:8080 rental-backend
```

**前端镜像**：
```bash
cd frontend
docker build -t rental-frontend .
docker run -p 3000:80 rental-frontend
```

## 7. 常见问题排查

### 7.1 数据库连接失败

**问题现象**：后端启动时报错 `Cannot connect to database`

**排查步骤**：
1. 检查 MySQL 服务是否启动：`systemctl status mysql`
2. 检查数据库配置：确认 `application.yml` 中的用户名密码正确
3. 检查端口：确认 MySQL 默认端口 3306 未被占用
4. 检查防火墙：确保 3306 端口可访问

### 7.2 前端无法访问后端

**问题现象**：前端请求显示"网络错误"

**排查步骤**：
1. 检查后端服务是否启动：访问 `http://localhost:8080/api/health`
2. 检查跨域配置：确认后端允许前端域名
3. 检查端口配置：确认前端 `VITE_API_BASE_URL` 配置正确
4. 检查防火墙：确保 8080 端口可访问

### 7.3 登录失败

**问题现象**：登录时提示"验证码错误"或"登录失败"

**排查步骤**：
1. 检查验证码是否过期（5分钟）
2. 检查手机号是否正确
3. 检查 Redis 服务是否正常运行
4. 查看后端日志：确认验证码是否正确存储

### 7.4 图片无法显示

**问题现象**：房源图片显示为占位符

**排查步骤**：
1. 检查后端返回的图片 URL 是否有效
2. 检查图片服务器是否可访问
3. 检查网络连接

### 7.5 Docker 启动失败

**问题现象**：`docker-compose up` 启动失败

**排查步骤**：
1. 检查端口是否被占用：`netstat -tlnp | grep 3306`
2. 检查 Docker 服务是否启动：`systemctl status docker`
3. 查看容器日志：`docker logs rental-backend`

## 8. 附录

### 8.1 API 接口列表

| 接口 | 方法 | 认证 | 说明 |
|------|------|------|------|
| `/auth/send-code` | POST | 否 | 发送验证码 |
| `/auth/login` | POST | 否 | 手机号登录 |
| `/auth/refresh` | POST | 是 | 刷新令牌 |
| `/user/info` | GET | 是 | 获取用户信息 |
| `/user/info` | PUT | 是 | 更新用户信息 |
| `/house/page` | GET | 否 | 分页查询房源 |
| `/house/{id}` | GET | 否 | 房源详情 |
| `/house/recommend` | GET | 否 | 推荐房源 |
| `/news/page` | GET | 否 | 分页查询新闻 |
| `/news/{id}` | GET | 否 | 新闻详情 |
| `/region/list` | GET | 否 | 区域列表 |

### 8.2 状态码说明

| 状态码 | 说明 |
|--------|------|
| 200 | 操作成功 |
| 400 | 请求参数错误 |
| 401 | 未认证或令牌过期 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |
