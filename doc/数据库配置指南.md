## 1. 问题分析

根据 ERROR.txt 日志，主要问题是：

```
java.sql.SQLException: Access denied for user 'root'@'localhost' (using password: YES)
```

**错误原因**：MySQL 数据库密码认证失败，当前配置的密码不正确。

---

## 2. 解决方案

### 方案一：使用现有数据库账户（推荐）

如果您已经有 MySQL 账户，修改配置文件中的密码即可。

### 方案二：使用默认密码 `root`

如果您的 MySQL root 用户密码是 `root`，直接修改配置。

### 方案三：设置新的 MySQL 密码

如果忘记了密码，可以重置 MySQL root 密码。

---

## 3. 配置步骤

### 步骤 1：确认 MySQL 服务状态

确保 MySQL 服务已启动：

```bash
# Windows (以管理员身份运行)
net start mysql80  # 或 mysql

# Linux/Mac
sudo systemctl start mysql
# 或
sudo service mysql start
```

### 步骤 2：确认您的 MySQL 密码

尝试登录 MySQL：

```bash
mysql -u root -p
# 输入密码后按回车
```

如果能成功登录，说明密码正确。

### 步骤 3：修改 application.yml 配置

编辑 `backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/rental_system?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
    username: root                    # 修改为你的 MySQL 用户名
    password: your_password_here       # 修改为你的 MySQL 密码
```

### 步骤 4：创建并初始化数据库

如果数据库还不存在，执行初始化脚本：

```bash
# 登录 MySQL
mysql -u root -p

# 执行初始化脚本（在 MySQL 命令行中）
source e:/Workspace/Java/GreatCreate/sql/init.sql;

# 或者在命令行直接执行
mysql -u root -p < e:/Workspace/Java/GreatCreate/sql/init.sql
```

## 5. 常见问题

### Q1：忘记了 MySQL root 密码怎么办？

**Windows**：

```bash
# 1. 停止 MySQL 服务
net stop mysql80

# 2. 以安全模式启动 MySQL
mysqld --skip-grant-tables --console

# 3. 在另一个命令行窗口登录
mysql -u root

# 4. 重置密码
USE mysql;
FLUSH PRIVILEGES;
ALTER USER 'root'@'localhost' IDENTIFIED BY 'your_new_password';
FLUSH PRIVILEGES;
EXIT;

# 5. 重启 MySQL 服务
net start mysql80
```

**Linux/Mac**：

```bash
# 1. 停止 MySQL 服务
sudo systemctl stop mysql

# 2. 以安全模式启动
sudo mysqld_safe --skip-grant-tables &

# 3. 登录并重置密码
mysql -u root
USE mysql;
FLUSH PRIVILEGES;
ALTER USER 'root'@'localhost' IDENTIFIED BY 'your_new_password';
FLUSH PRIVILEGES;
EXIT;

# 4. 重启服务
sudo systemctl start mysql
```

### Q2：数据库 `rental_system` 不存在怎么办？

执行初始化脚本即可：

```bash
mysql -u root -p < sql/init.sql
```

### Q3：连接时提示 `Public Key Retrieval is not allowed`

在 JDBC URL 中添加 `allowPublicKeyRetrieval=true`（已在配置文件中添加）。

### Q4：端口 3306 被占用怎么办？

检查端口占用：

```bash
netstat -ano | findstr "3306"
```

如果端口被占用，可以：
1. 停止占用该端口的程序
2. 修改 MySQL 配置使用其他端口
3. 修改 application.yml 中的端口号
